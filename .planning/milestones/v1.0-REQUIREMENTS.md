# Requirements: UIUX-Mirror (uidna)

**Defined:** 2026-02-15
**Core Value:** Every extracted standard and inferred component must trace back to observable evidence from the source site

## v1 Requirements (ALL COMPLETE)

### Crawling & Compliance

- [x] **CRAWL-01**: User can crawl a site section (20-100 pages) from seed URLs with configurable depth
- [x] **CRAWL-02**: Crawler respects robots.txt with configurable strict mode
- [x] **CRAWL-03**: Crawler enforces rate limiting and request throttling
- [x] **CRAWL-04**: Crawler handles dynamic content (CSS-in-JS wait strategies, framework-specific markers)
- [x] **CRAWL-05**: Crawler uses anti-bot stealth configuration (user-agent rotation, timing jitter)
- [x] **CRAWL-06**: User can re-crawl a site and see diff of what changed since last crawl

### Token Extraction

- [x] **TOKEN-01**: Extract color palette with semantic grouping (primary, secondary, accent, neutral)
- [x] **TOKEN-02**: Extract typography scale (families, sizes, weights, line-heights)
- [x] **TOKEN-03**: Extract spacing/sizing scale with base unit detection
- [x] **TOKEN-04**: Parse CSS custom properties and map to token categories
- [x] **TOKEN-05**: Extract border radii, elevation/shadow, and z-index scales
- [x] **TOKEN-06**: Extract motion/animation tokens (durations, easings, keyframes)
- [x] **TOKEN-07**: Analyze icon style (stroke weight, fill patterns, size conventions)
- [x] **TOKEN-08**: Detect imagery patterns (aspect ratios, treatment styles, placeholder conventions)

### Component Mining

- [x] **COMP-01**: Identify common components from DOM structures (buttons, inputs, cards, nav, modals)
- [x] **COMP-02**: Detect variant dimensions per component (size, color, emphasis)
- [x] **COMP-03**: Map interactive states per component (hover, active, focus, disabled, loading, error)
- [x] **COMP-04**: Score component consistency across pages (statistical confidence)

### Pattern Detection

- [x] **PATT-01**: Detect multi-page interaction patterns (auth flows, checkout, onboarding, search/filter)
- [x] **PATT-02**: Extract content/microcopy rules (voice/tone, CTA hierarchy, capitalization, error message grammar)

### Normalization & Evidence

- [x] **NORM-01**: Deduplicate extracted values with fuzzy matching (color distance, unit normalization)
- [x] **NORM-02**: Output tokens in standard JSON format (W3C Design Token spec compatible)
- [x] **NORM-03**: Store evidence per observation (page URL, DOM selector, screenshot crop, computed styles, timestamp)
- [x] **NORM-04**: Enforce cross-page validation thresholds before declaring a value a "standard"
- [x] **NORM-05**: Assign confidence scores to every extracted token and component

### Inference Engine

- [x] **INFER-01**: Synthesize unknown component structure using rule-based composition of observed primitives
- [x] **INFER-02**: Use Claude API for nuanced design judgment (motion, edge states, copy decisions)
- [x] **INFER-03**: Emit confidence score and evidence citations for every inference
- [x] **INFER-04**: Generate complete state coverage (hover, active, focus, disabled, loading, error)
- [x] **INFER-05**: Generate accessibility baseline per synthesized component (keyboard, focus, ARIA guidance)

### Reporting & Export

- [x] **EXPORT-01**: Generate human-readable Brand DNA Report (Markdown)
- [x] **EXPORT-02**: Export tokens.json (colors, typography, spacing, radii, shadows, z-index, motion)
- [x] **EXPORT-03**: Export components.json (observed components, variants, states)
- [x] **EXPORT-04**: Export patterns.json (detected interaction flows)
- [x] **EXPORT-05**: Export content_style.json (microcopy rules and examples)
- [x] **EXPORT-06**: Export evidence_index.json (URLs, DOM selectors, screenshots, timestamps, confidence)
- [x] **EXPORT-07**: Generate framework-agnostic component stubs (vanilla HTML/CSS)
- [x] **EXPORT-08**: Support multiple export formats (CSS variables, Tailwind config, Figma tokens)

### CLI & MCP Interface

- [x] **CLI-01**: `uidna crawl <url>` command with depth, page limit, and domain allowlist options
- [x] **CLI-02**: `uidna extract` command to run extraction pipeline on crawled data
- [x] **CLI-03**: `uidna report` command to generate Brand DNA Report
- [x] **CLI-04**: `uidna synth <component>` command to synthesize a new component
- [x] **CLI-05**: `uidna export` command with format flag (tokens, figma, tailwind, cssvars, json)
- [x] **CLI-06**: MCP server mode exposing design DNA as queryable tools/resources
- [x] **CLI-07**: Configuration via `uidna.config.json` (depth, limits, allowlist/denylist, strict robots, viewport sizes)
